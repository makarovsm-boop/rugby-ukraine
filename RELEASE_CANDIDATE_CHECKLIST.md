# Release Candidate Checklist

Короткий фінальний checklist перед першим публічним запуском MVP без самого deploy.

## 1. Що вже можна вважати готовим

До першого публічного запуску вже готові:

- публічні розділи сайту:
  - головна
  - новини
  - матчі
  - чемпіонати
  - команди
  - гравці
  - для новачків
  - про проєкт
- пошук по основних сутностях
- авторизація
- профіль користувача
- коментарі під статтями
- базова адмінка
- ролі `USER / EDITOR / ADMIN`
- SEO-база:
  - metadata
  - sitemap
  - robots
  - базовий structured data
- production prep docs
- mobile-first і UX-polish pass

## 2. Фінальний короткий checklist

Перед першим запуском пройти:

- `env` заповнені реальними production-значеннями
- `DATABASE_URL` вказує на production БД
- `NEXTAUTH_URL` збігається з реальним доменом
- `NEXT_PUBLIC_SITE_URL` збігається з реальним доменом
- `NEXTAUTH_SECRET` замінено на реальний безпечний секрет
- `NEXT_PUBLIC_GA_ID` додано, якщо аналітика потрібна одразу
- `prisma generate` проходить
- `build` проходить
- `start` проходить у production-like середовищі
- створено реальний `ADMIN`
- створено окремий `EDITOR`
- тестові акаунти та seed-контент не лишаються єдиним сценарієм роботи

## 3. Що обов'язково перевірити вручну

### Public flow

- Головна сторінка відкривається без помилок.
- Новини відкриваються списком і окремими сторінками.
- Матчі відкриваються списком і detail-сторінками.
- Переходи:
  - головна -> новини
  - головна -> матчі
  - команда -> гравець
  - матч -> команда
  - пошук -> сутність
- `robots.txt` доступний.
- `sitemap.xml` доступний.

### Auth flow

- Вхід працює.
- Вихід працює.
- `callbackUrl` після логіну працює правильно.
- `USER` не бачить адмінку.
- `EDITOR` бачить лише статті й коментарі.
- `ADMIN` має повний доступ.

### Admin flow

- `ADMIN` може:
  - створити статтю
  - зберегти статтю як `Draft`
  - опублікувати статтю
  - відредагувати статтю
  - видалити статтю
- `EDITOR` може:
  - створити й редагувати статті
  - модерувати коментарі
- Форми не ламаються на mobile.
- Завантаження зображень працює.
- Fallback-зображення працює.

### Search flow

- Пошук знаходить:
  - новини
  - команди
  - гравців
  - чемпіонати
- Результати мають зрозумілий тип сутності.
- Порожній запит і пустий результат виглядають адекватно.

## 4. Minimum Content Gate

Перед першим публічним запуском бажано, щоб у сайту вже було:

- щонайменше `3-5` реальних новин
- щонайменше `1` реальна редакційна чернетка
- базово наповнені команди й гравці
- кілька матчів із різними статусами
- прибрані найбільш очевидні demo-заглушки з публічних екранів

## 5. Що не повинно блокувати перший запуск

Для першого запуску не обов'язково мати:

- складний WYSIWYG-редактор
- зовнішній media CDN
- recovery flow для пароля
- складну ACL-систему
- production-grade analytics dashboard
- повний перехід усіх даних на ідеально реальний контент

Якщо базові user flows стабільні, MVP уже можна відкривати публічно.

## 6. Release Candidate Verdict

Поточний стан проєкту можна вважати `release candidate`, якщо одночасно виконані три умови:

1. Production env заповнені коректно.
2. Public, auth, admin і search flows пройдені вручну без критичних помилок.
3. На сайті вже є мінімальний набір реального контенту, а не лише seed-дані.

Якщо ці умови виконані, проєкт готовий до першого публічного запуску MVP.
