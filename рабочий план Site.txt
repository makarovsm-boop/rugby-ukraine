Ниже — рабочий план, чтобы делать проект через Codex не вслепую, а как нормальный медиасайт про регби для украинской аудитории.

Смотрю на твой образец: у Ruck главная логика — это новостной сайт с сильной рубрикацией: последние новости, крупные турниры, клубы, женское регби, отдельные категории, поиск, трендовые материалы и постоянный поток свежих публикаций. У них меню и структура построены именно вокруг чемпионатов, команд и новостей, а не просто вокруг “страниц о спорте”. Это хороший ориентир для твоего проекта. ([Ruck][1])

## 1. Идея проекта

Тебе нужен не просто “сайт про регби”, а **украинская медиа-платформа и сообщество**:

* новости о регби
* страницы чемпионатов
* страницы игроков и команд
* календарь матчей и результаты
* объяснение правил для новичков
* площадка для общения болельщиков
* контент для популяризации регби в Украине

То есть сайт должен решать сразу 3 задачи:

1. **Информировать** — новости, обзоры, анонсы, результаты
2. **Объяснять** — что такое регби, турниры, позиции, правила
3. **Объединять** — комментарии, обсуждения, сообщество

Вот это уже звучит как проект, а не как “еще один блог на три статьи”.

## 3. Структура сайта

### Главная страница

На главной должно быть:

* большой главный материал дня
* последние новости
* ближайшие матчи
* результаты последних матчей
* блок “что такое регби” для новых людей
* популярные игроки / команды
* обсуждаемые темы
* блок “регби в Україні”
* подписка на новости / Telegram / соцсети

### Новости

Фильтры:

* по турниру
* по стране
* по клубу
* по сборной
* по женскому регби
* по тегам

### Чемпионаты

Для каждого турнира отдельная страница:

* описание турнира
* формат
* таблица
* расписание
* результаты
* новости по турниру

Примеры:

* Six Nations
* Rugby World Cup
* Premiership Rugby
* URC
* Top 14
* EPCR / Champions Cup
* регби в Украине

### Игроки

Карточка игрока:

* имя
* фото
* страна
* позиция
* клуб / сборная
* краткая биография
* достижения
* последние новости о нем
* связанные материалы

### Команды

Карточка команды:

* логотип
* страна
* лига
* состав
* последние матчи
* новости
* история клуба

### Для новичков

Очень важный раздел для популяризации:

* что такое регби
* чем rugby union отличается от rugby league
* как начисляются очки
* позиции игроков
* что такое scrum, lineout, try, conversion
* как смотреть матч и понимать, что происходит

Это один из самых недооценённых разделов. Он может привести много новой аудитории из Google.

### Сообщество

Минимум:

* комментарии под статьями
* личный кабинет
* профиль пользователя
* лайки / ответы

Потом можно добавить:

* форум
* фан-клубы по командам
* рейтинги пользователей
* пользовательские посты

## 4. Что взять из ruck.co.uk как ориентир

У Ruck хорошо видно несколько сильных вещей:

* сильная новостная лента
* разбивка по турнирам и клубам
* заметные trending/top news блоки
* много категорий в меню
* постоянный поток контента ([Ruck][1])

Но тебе **не нужно копировать их полностью**. Лучше взять их как образец **архитектуры контента**, а не дизайна один в один.

Что стоит перенять:

* понятное меню
* новости на главной
* рубрики по турнирам
* страницы команд / игроков
* трендовые материалы
* поиск

Что стоит улучшить под украинскую аудиторию:

* украинский язык по умолчанию
* блок “регби для новичков”
* акцент на украинское регби
* простые объяснения терминов
* комьюнити и общение

## 6. Технологический стек для Codex

Если хочешь делать через Codex, я бы рекомендовал такой стек:

### Вариант, который реально удобен:

* **Next.js**
* **TypeScript**
* **Tailwind CSS**
* **PostgreSQL**
* **Prisma**
* **NextAuth / Clerk** для авторизации
* **Sanity / Strapi / Payload / Supabase** — для контента
* **Vercel** — для деплоя

### Почему так:

* Next.js хорош для SEO
* TypeScript поможет не развалить проект
* Tailwind быстро даёт нормальный UI
* PostgreSQL нормален для новостей, игроков, команд, комментариев
* CMS нужна, чтобы ты публиковал статьи без боли

### Самый практичный вариант:

**Next.js + Supabase + Tailwind + Prisma**
или
**Next.js + Sanity + Supabase**

## 8. Контент-стратегия

Вот тут часто и ломаются хорошие идеи. Сайт есть, а писать нечего.

Нужны 4 типа контента:

### 1. Новости

* трансферы
* результаты
* анонсы матчей
* травмы
* заявления тренеров
* составы

### 2. Вечнозелёный контент

* правила регби
* лучшие игроки
* история турниров
* как начать смотреть регби
* базовые термины

### 3. Аналитика

* разбор матчей
* тактика
* сравнение игроков
* перспективы команд

### 4. Социальный контент

* опросы
* обсуждения
* рейтинги
* “символическая сборная тура”

### Для запуска подготовь сразу:

* 10–15 базовых статей “для новичков”
* 10 карточек игроков
* 10 карточек команд
* 5 страниц чемпионатов
* 15–20 новостей / материалов

Пустой сайт выглядит грустно. Даже если дизайн красивый, без контента это просто дорогой шкаф без одежды.

## 9. UX и дизайн советы

Для украинской аудитории я бы делал стиль:

* современный
* спортивный
* чистый
* не перегруженный
* акцент на читаемость

### Что важно:

* крупный заголовок
* хорошие карточки новостей
* заметные блоки турниров
* удобное меню
* адаптация под телефон
* быстрый поиск
* тёмная тема — можно позже

### Цвета

Подойдут:

* тёмно-зелёный / графит / белый
* бордовый / тёмно-синий как акценты
* не делать слишком кислотно

## 10. SEO и рост аудитории

Для такого проекта SEO — золотая жила.

### Что важно сделать сразу:

* ЧПУ-ссылки
  например:
  `/news/england-win-six-nations`
  `/players/antoine-dupont`
  `/championships/six-nations`

* meta title / description

* Open Graph для соцсетей

* schema markup для статей

* sitemap.xml

* robots.txt

* быстрый сайт

* alt-тексты изображений

### Какие запросы будут полезны:

* что такое регби
* правила регби
* чемпионат шести наций
* регби украина
* игроки сборной [страна]
* расписание матчей регби
* результаты регби

## 13. Как ставить задачи Codex

Чтобы Codex не делал кашу, лучше разбивать проект на маленькие понятные задачи.

Примеры хороших задач:

* “Создай Next.js проект с TypeScript и Tailwind для новостного сайта о регби”
* “Сделай главную страницу с hero-блоком, лентой новостей и блоком турниров”
* “Создай Prisma schema для articles, players, teams, championships, comments”
* “Сделай страницу статьи с SEO metadata”
* “Сделай страницу игрока с фото, биографией и связанными новостями”
* “Сделай форму комментариев для авторизованных пользователей”
* “Подключи Supabase auth”
* “Сделай админскую страницу для создания статьи”

### Главное правило:

не просить “сделай весь сайт”.
Иначе Codex может устроить творческий джаз, а ты потом будешь ловить его по всему проекту.

## 15. Мои практические советы

### 1. Не начинай с форума

Форум без аудитории — как стадион без зрителей. Сначала контент.

### 2. Сначала сделай украинский язык основным

Потом можно добавить английский или русский, но ядро лучше сразу делать украинским.

### 3. Делай упор на объяснение регби

Это поможет популяризации, а не только существующим фанатам.

### 4. Сразу продумай стиль публикаций

Нужны шаблоны:

* новость
* анонс матча
* обзор матча
* биография игрока
* страница турнира

### 5. Не перегружай главную

Лучше меньше блоков, но понятнее.

### 6. Подумай над брендом

Название, логотип, tone of voice, визуальный стиль — это важно для узнаваемости.

### 7. Используй источники аккуратно

Если будешь брать данные о матчах, игроках, фото и т.д., сразу думай о лицензиях и источниках.
