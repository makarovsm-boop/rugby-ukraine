# Post-Launch UX Checklist

## 1. What to Watch After First Real Users

Після перших користувачів у першу чергу перевірити:

1. чи люди розуміють, з чого почати на головній
2. чи переходять із головної в `Новини`, `Матчі`, `Для новачків`
3. чи не губляться між `Команди`, `Гравці`, `Чемпіонати`
4. чи помітний пошук у шапці
5. чи зрозуміло, як потрапити в адмінку після входу

## 2. Navigation Weak Spots

Найімовірніші слабкі місця навігації:

- `header` на мобільному: чи всі пункти читаються і не виглядають перевантажено
- перехід між `Команда -> Гравець -> Команда`
- перехід між `Чемпіонат -> Матч -> Команди`
- видимість CTA на головній сторінці
- розуміння різниці між `Новини` і `Матчі`

Що дивитися вручну:

1. скільки кліків потрібно, щоб дійти від головної до потрібного матеріалу
2. чи є сторінки, де користувач не бачить очевидного наступного кроку
3. чи не надто багато однакових кнопок `Перегляд / Детальніше / Відкрити`

## 3. Mobile UX Risk Areas

Найризикованіші місця для mobile:

- шапка з навігацією і пошуком
- hero та великі картки на головній
- списки новин із великими картками
- сторінки команди і чемпіонату з кількома блоками підряд
- таблиця на сторінці чемпіонату
- сторінка матчу з рахунком і двома командами
- адмінські форми з великою кількістю полів

Що перевірити на телефоні:

1. чи не зламуються великі заголовки у 2–4 рядки
2. чи не занадто великі вертикальні відступи між картками
3. чи не втрачається головна дія кнопки після першого екрану
4. чи зручно натискати filter pills
5. чи не занадто довгі форми без явних візуальних пауз

## 4. Pages to Recheck First

Після запуску вручну пройтись по цих сторінках:

- `/`
- `/news`
- `/news/[slug]`
- `/matches`
- `/matches/[id]`
- `/championships`
- `/championships/[slug]`
- `/teams`
- `/teams/[slug]`
- `/players`
- `/players/[slug]`
- `/search`
- `/login`
- `/profile`

## 5. Content UX Questions

Питання, на які варто відповісти після перших користувачів:

1. чи достатньо просто написані тексти для новачка
2. чи не виглядають блоки занадто “редакційно”, якщо людина прийшла лише подивитися матчі
3. чи видно український фокус сайту без додаткових пояснень
4. чи не перевантажені сторінки другорядною інформацією

## 6. Admin UX Checks

Після перших редакторів або адміна перевірити:

1. чи зрозумілий `draft / published` flow
2. чи достатньо ясно, яке зображення реально буде показано
3. чи не плутають create- і edit-сторінки
4. чи легко повернутися зі сторінки редагування до списку
5. чи не перевантажені довгі форми

## 7. Signals That UX Needs Immediate Fixes

Ознаки, що треба швидко повертатися до UX-polish:

- користувачі не доходять до ключових розділів із головної
- часто використовують back navigation замість явних посилань
- плутають `Команди` і `Гравці`
- не помічають пошук
- на mobile сторінки здаються “занадто довгими”
- в адмінці часто створюють контент не в тому статусі

## 8. Practical Review Rhythm

Після запуску зручно працювати так:

1. у перший день перевірити основні сценарії вручну
2. за 3–7 днів зібрати повторювані UX-спостереження
3. окремо виписати проблеми:
   - navigation
   - mobile
   - content clarity
   - admin UX
4. виправляти спочатку ті речі, які заважають дістатися до контенту або завершити дію
