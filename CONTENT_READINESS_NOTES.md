# Real Content Readiness Notes

Короткий pass перед переходом від seed-даних до реального редакційного наповнення.

## 1. Editor Admin Flow

Що вже готово:

- `Draft / Published` flow для статей працює.
- `EDITOR` має доступ лише до статей і коментарів.
- Є create/edit/delete для статей.
- Є moderation flow для коментарів.
- Є окремі success/error стани у формах.

Що перевірити редактору перед реальною роботою:

- Створити окремий акаунт з роллю `EDITOR` у `/admin/users` і пройти повний flow:
  - створення чернетки
  - редагування
  - публікація
  - повернення в чернетку
  - модерація коментарів
- Переконатися, що редактору не потрібен доступ до команд, гравців, матчів і користувачів.

Практичне зауваження:

- У seed зараз є тестовий `ADMIN`, але окремого seed-акаунта для `EDITOR` немає.
- Для реальної роботи редакції краще створити окремого `EDITOR` через `/admin/users`, а не працювати з головного адмінського акаунта.

## 2. Media Flow

Що вже готово:

- Для статей, команд, гравців і чемпіонатів є:
  - ручний шлях до зображення
  - локальне завантаження файла
  - прев'ю в адмінці
  - fallback-зображення
- Якщо шлях порожній або файл не існує, сайт не ламається.
- Зображення зберігаються локально в `public/uploads`.

Що перевірити перед реальним контентом:

- Чи підходить редакції локальний сценарій зберігання файлів у `public/uploads`.
- Чи влаштовує поточний набір форматів: `jpg`, `png`, `webp`, `svg`.
- Чи редактори розуміють різницю між:
  - ручним шляхом у `public`
  - локальним upload через форму
  - fallback-сценарієм

Практичне зауваження:

- Для production-контенту краще домовитися про єдину дисципліну назв файлів:
  - `articles/...`
  - `teams/...`
  - `players/...`
  - `championships/...`
- Бажано не змішувати випадкові імена файлів і ручні шляхи без узгодженого правила.

## 3. Places That Still Feel Too Test-Like

Ось що варто замінити перед реальним запуском контенту:

- Seed-користувачі з email на `*.local`:
  - це нормально для dev,
  - але для реальної редакції потрібні реальні робочі акаунти.
- Тестова чернетка статті:
  - `draft-ukraine-rugby-calendar`
  - це службовий seed-матеріал, не реальний редакційний контент.
- Частина seed-зображень:
  - зараз це акуратні локальні SVG/mock assets,
  - для живого сайту їх варто поступово замінити на реальні фото або більш редакційно правдоподібні візуали.
- Production env приклади:
  - `rugby-ukraine.example.com`
  - `G-XXXXXXXXXX`
  - це лише шаблони, не фінальні значення.

## 4. Content Priority Before Real Fill

Найпрактичніший порядок для переходу на реальний контент:

1. Створити окремий редакторський акаунт `EDITOR`.
2. Замінити seed-чернетку на справжню робочу чернетку редакції.
3. Наповнити 3-5 реальних статей через адмінку, а не через seed.
4. Замінити головні зображення для новин і команд на реальні.
5. Перевірити, як виглядають:
   - `/news`
   - `/news/[slug]`
   - `/teams`
   - `/players`
   - `/championships`
   уже на живому редакційному контенті.

## 5. Readiness Verdict

Поточний стан:

- Архітектурно проєкт готовий до реального наповнення контентом.
- Редакторський flow для MVP достатній.
- Media flow для MVP достатній.
- Основний remaining risk не технічний, а редакційний:
  - тестові seed-дані
  - тимчасові demo-акаунти
  - частина ілюстрацій ще виглядає як MVP/mock content.
